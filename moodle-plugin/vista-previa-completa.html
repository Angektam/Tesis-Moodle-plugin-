Resumen del Sistema

¿Qué es AI Assignment?
Un plugin de Moodle que permite a los profesores crear tareas que son evaluadas automáticamente usando Inteligencia Artificial (OpenAI GPT), proporcionando retroalimentación instantánea a los estudiantes.

Evaluación Instantánea

Los estudiantes reciben feedback inmediato al enviar sus respuestas

IA Avanzada

Usa GPT-4o-mini para comparar y evaluar respuestas de forma inteligente

Dashboard Visual

Panel de control moderno con estadísticas y gráficas en tiempo real

GDPR Compliant

Cumple con regulaciones de privacidad y protección de datos

Flujo del Sistema

1. Profesor Crea la Tarea

El profesor accede a su curso en Moodle, activa la edición y agrega una nueva actividad "AI Assignment". Configura el nombre, descripción, tipo de problema (matemáticas o programación), la solución de referencia, documentación adicional, casos de prueba y parámetros de calificación.

2. Estudiante Ve la Tarea

El estudiante entra al curso y hace clic en la actividad AI Assignment. Ve la descripción del problema, la documentación proporcionada, casos de prueba de ejemplo y un formulario para enviar su respuesta. También puede ver sus envíos anteriores y el número de intentos restantes.

3. Estudiante Envía Respuesta

El estudiante escribe su solución en el área de texto y hace clic en "Enviar". El sistema valida que la respuesta no esté vacía, verifica que no haya excedido el máximo de intentos permitidos, y guarda el envío en la base de datos con estado "pending".

4. Evaluación con IA

El sistema llama automáticamente al evaluador de IA (ai_evaluator.php) que prepara dos prompts: uno de sistema (define el rol de la IA) y uno de usuario (contiene ambas soluciones). Envía la petición a la API de OpenAI GPT-4o-mini con temperatura 0.3 para respuestas consistentes.

5. IA Compara y Analiza

OpenAI GPT analiza ambas soluciones según criterios específicos: para código evalúa funcionalidad, estilo, buenas prácticas y eficiencia; para matemáticas evalúa corrección, método, claridad y pasos. Genera un similarity_score (0-100), feedback breve y análisis detallado en formato JSON.

6. Guardar Resultados

El sistema recibe la respuesta JSON de OpenAI, valida el formato, extrae los datos y los guarda en la tabla aiassignment_evaluations. Actualiza el envío con el score y feedback, cambia el estado a "evaluated" y actualiza la calificación en el libro de calificaciones de Moodle.

7. Estudiante Ve Resultados

El estudiante ve inmediatamente su calificación (0-100%), la retroalimentación breve de la IA y puede hacer clic en "Ver detalles" para acceder al análisis completo. Si tiene intentos restantes, puede enviar una nueva respuesta mejorada.

8. Profesor Monitorea

El profesor puede acceder al dashboard del curso para ver estadísticas globales, resumen de todas las tareas, envíos recientes y top performers. También puede ver todos los envíos de una tarea específica, revisar evaluaciones individuales y re-evaluar si es necesario.

Evaluación con Inteligencia Artificial

Proceso de Evaluación

1. Preparación de Prompts

System Prompt: Define el rol de la IA como evaluador experto

Eres un asistente experto en evaluación de código de programación. Tu tarea es comparar la respuesta de un estudiante con la solución del profesor y proporcionar una evaluación justa y constructiva. Debes responder ÚNICAMENTE en formato JSON con esta estructura exacta: {"similarity_score": número entre 0 y 100, "feedback": "texto breve", "analysis": "análisis detallado"}

User Prompt: Contiene ambas soluciones y criterios

Compara estas dos soluciones de programación: SOLUCIÓN DEL PROFESOR: def suma(a, b): return a + b RESPUESTA DEL ESTUDIANTE: def sumar_numeros(num1, num2): resultado = num1 + num2 return resultado Evalúa: 1. Funcionalidad (¿hace lo que debe hacer?) 2. Estilo y claridad del código 3. Buenas prácticas 4. Eficiencia

2. Llamada a OpenAI API

Configuración:

  • Endpoint: https://api.openai.com/v1/chat/completions
  • Modelo: gpt-4o-mini (rápido y económico)
  • Temperature: 0.3 (respuestas consistentes)
  • Response format: json_object (garantiza JSON válido)

3. Análisis de la IA

OpenAI GPT analiza ambas soluciones según los criterios especificados y genera una respuesta estructurada.

4. Respuesta JSON

{ "similarity_score": 92, "feedback": "Excelente solución. El código es correcto y bien documentado. Los nombres de variables son descriptivos.", "analysis": "Funcionalidad: ✓ Perfecto\nEstilo: ✓ Muy bueno\n Buenas prácticas: ✓ Bueno\nEficiencia: ✓ Óptimo" }

Criterios de Evaluación

Tipo Criterio Peso Qué Evalúa
Programación Funcionalidad 40% ¿Funciona correctamente? ¿Produce el resultado esperado?
Estilo y Claridad 25% Nombres descriptivos, indentación, comentarios, legibilidad
Buenas Prácticas 20% Validación, manejo de errores, código modular, SOLID
Eficiencia 15% Complejidad temporal, uso de memoria, optimización
Matemáticas Corrección 50% ¿La respuesta es correcta? ¿El resultado es el esperado?
Método 25% ¿El método usado es apropiado? ¿Es eficiente?
Claridad 15% ¿La explicación es clara? ¿Se entiende el razonamiento?
Pasos 10% ¿Se muestran los pasos? ¿El proceso es lógico?
